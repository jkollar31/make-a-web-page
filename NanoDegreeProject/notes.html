Web Pages, Documents, and Structure

Lesson 1: The Basics of Web and HTML

Web and HTML

With a computer and a browser a person can send a request to a website using the HTTP protocol and the server will send back to your computer through a browser the HTML documents that lets the person view the web page. Using HTML documents we can display and share information across the web. HTML documents are made up of things like text, files, images, videos, and links to other webpages like reddit. In order to manipulate the information on an HTML documemt we use tags and attributes.

Tags can either be inline or block

Inline meaning the information just continues on wrapping around.
Block meaning the information is contained with in a block.

Lesson 2: Creating a Structured Document with HTML

How to Translate Information to a Webpage
  • HTML and CSS are both "languages"
  • HTML controls the "structure" of a web page.
    • Tags create elements in a tree or the "DOM"
    • Tags will contain content that can be empty, text or another element.
    • Tags can have types paragraph, image, or document division.
    • Tags can have attributes with values.
  • CSS controls the "style" of a page(how it looks).
    • CSS changes how the elements look on a page font, color, positioning, etc...
  • When programmers talk about the "DOM" they are talking about the tree like structure of the page.
    • A tree structure is the hierarchical nature of the structure in graphical form. The structure is upside down with the root at the top and the leaves expanding down from the root.
Grids, Boxes, and Web Pages
  • Web pages are made up of grids
    • Boxes are aligned using the grids.
      • Elements are contained within the boxes.
  • Create boxes to start a web page
    • By creating boxes you can work from the big to the small. Start with the big boxes and work down through the structure creating smaller boxes.
    • Use class names to label the boxes.

Lesson 3: Adding CSS Style to HTML Structure

CSS Structure

Two ways to select an HTML element:

  • Option 1: By element tag... EX. h1{}
  • Option 2: By element class... EX. .class-name{}
Adding style to HTML

Option 1:

  1. Add a head tag
  2. Add a link tag inside of head tag
  3. Add the file location of CSS file

Option 2:

  1. Add a style tag
  2. Insert CSS between style tag

Option 3:(Not Recommended)

  1. Add stlye attribute to individual elements
Inheritance

Inheritance in CSS is the mechanism by which properties are applied to a specified element, but also to its descendants. Inheritance prevents repetition of certain properties being declared.

Properties that can be inherited are:

  • color, font, letter-spacing, line-height, list-style, text-align text-indent, text-transform, visibility, white-space and word-spacing.

Properties that cannot be inherited are:

  • background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.
The Box Model

The size of the actual element is equal to the sum of the border, padding and content width

  • An easier way to calculate the size is to use CSS styling box-sizing: border-box
    • Just set the width of the overall box, so size will always be the same no matter how to change the padding or border
    • In order for older browser to be able to display this style you need to add browser specific prefixes:
      • -webkit
      • -moz
      • -ms
  • The size of a box can either be by pixel or percentage(recommended)

Positioning of Boxes

  • The Flex box layout means a flexable box
  • Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  • Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
Code, Test, Refine
  1. Do this by visually boxifying the structure.
  2. Look for repeated Styles and semantic elements
  3. Write your HTML
  4. Apply Styles (from biggest to smallest)
  5. Fix small things
  6. Repeat steps 3-4 until happy
